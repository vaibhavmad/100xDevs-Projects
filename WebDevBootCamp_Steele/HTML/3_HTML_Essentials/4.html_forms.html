HTML Forms

Form is a sort of a shell or a container, under which multiple elements are grouped under the 'Form' tags. These are individual input and button elements.

When we create form tags, we also get an attribute by the name of action - which primarily deals with where the data should be sent.

So when a form is submitted, a HTTP request is sent with the information that we are searching for and this information is then searched on the server and finally we get the result.

Form Elements

  1. Input Elements: This element has no closing tag and has an attribute - type (which specifies how the input will behave) and placeholder (basically the text that is mentioned before user enters information).

    Input Types

    1. text: as the name specifies, takes text as input.
    2. password: exactly as above, however, hides the text being written.
    3. color: gives us a color picker.
    4. file: gives option to upload a file.
    5. number: takes a number as an input from a dropdown (depends upon browser).
    6. time: time is taken as an input.

    Note:

    1. There are a lot of input types and the above list is not exhaustive. There are other such as range, etc. that shall be covered later.
    2. Placeholders does not work for a few elements such as date, time, colorpicker etc. This is because there is no option to show text due to the structure of these elements or due to the design of these elements.

  2. Label: Text that defines the element details and is attached to the input section. If a user clicks on a label of the attached input, the input box becomes active. This is also good for the screen readers.

    For label to work correctly, we need to mention for attribute in the label tag and then that same for attribute should be added as a id attribute in the input tag.

  3. Button: Buttons are common across applications, such as login, signup, search, and there are links that also look like a button. A button has a opening and closing tag. Buttons basically submit the form. Triggers the action basically. Otherwise not specified, by default the buttons simply submit the form, even if we have 3 buttons. Similarly to inputs, we have types in buttons. These are:
    1. button: This type behave like a simple button and not a submit button.
    2. submit: Type to submit the button.
    Note: Sometimes we will have input type as submit, not very regular. Here, we will have to add an attribute name value and under this we can decide on the text to be written on the button.
  4. Name: name is an input attribute, that tells where this data should be submitted to or queried from in the server. If we add a name, example username for input username and add value to username form and click on submit, we will see the information of username being submitted in the URL.

    This is basically the name that will be used to send data to the server. This must be added to all the input fields and will tell the server what this is data is all about.
    One thing that we noticed that the password is visible in the browser. This shall be changed later.
  5. Checkbox: Works on the basis of types only, with type being a checkbox. Also, we can a checkbox clicked by default or not. if we want the box to be clicked by default, we simply need to type checked in the tag, without inverted commas. In case of submission, if a checkbox is selected, data is sent as on.
  6. Radiobutton: Similar to checkboxes, however, in the given options, only one option can be selected. The type is simple: radio. To combine these buttons together, we need to give them the same name. However, to finalise which value should be submitted from the options in the radio, we need to add an attribute value. Value will not be displayed to the user, however, this shall be added to the information being sent to the server,
  7. Select Element (dropdown): Has a label, then select tag, followed by option tags containing all the options that we will have under the dropdown.
  8. Range & Text Area: Range is meant for selecting a range using a slider. text area is for taking a larger input from the user.
