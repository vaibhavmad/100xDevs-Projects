CSS Units

CSS units ar used to define the values around dimensions of various elements. So far we have majorly used px. As discussed initially, there are two types of units: Absolute and Relative.

Absolute: These units take up the given space independent of any other element, such as px, once defined shall take up the given space of the defined pixels. Some other similar units are inches, mm, cm, etc. Other than pixels, these are not used much.

Relative: Relative as the name suggests, are in relation to something else, most of the time to the parent element or in certain cases the element itself if we are trying to define an element based property such as line-height. The most commonly used relative units are: percentages, em and rem. Lets go through these one by one:

  1. Percentage: It is denoted using % sign and is relative to either the parent or the element. We shall cover both examples below.

    Let's first look at the parent - child percentage. We shall create a div inside another div and define width and height of the outer div in px and further define the inner in terms of the percentage. Here, the inner one shall take the sizing in relation to the dimensions of the outer one.

    .box1-outer
    background-color: aqua;
    width: 200px;
    height: 200px;
    .box1-inner
    background-color: blueviolet;
    width: 50%;
    height: 50%;

    .box2-outer
    background-color: aqua;
    width: 200px;
    height: 100px;
    .box2-inner
    background-color: blueviolet;
    width: 50%;
    height: 50%;

    As we can see above, both the inner boxes have been given the exact same dimensions, however, since they have relative units, their dimensions are in terms of the outer boxes and hence these are not equal to each other. We shall become handy with this with more practise. So, no point in spending a lot of time in getting the desired visual results. This is a simple and small part of being a developer. There are more important things so we shall focus on them.

    Advatanges of using Relative Units like em

    • This helps in a propotional development of the site. For example, if we are creating buttons by using px as the unit for everything, then while scaling up, the border, border radius, padding etc will remain as is even if the inner content font size is increased. However, if we do this using em, then just by changing the inner content font size, we can auto change the other elements as they are linked or are relative to the inner text.
    • Another benefit is that if we set a standard for one of the elements like body or an outer most div or any other suitable semantic tag, and keep rest of the elements relative to the same, then we can control the look and feel of the content by just changing that outer layer details and also again all the content will be similar and standard in nature in terms of the size as everything will be linked.
    These two are the reasons that on most websites, we see em or % as the main units.

  3. REM: While 'EM' has its advatages, it does comes with certain disadvantages as well. Let us first look at the disadvatage.

    When we talk about em, we know that each child will inherit dimensions from the parent and then shall convert it on the basis of n em units. If this n is 2, then the child will be twice the size of the parent. While this is great and helps in improving efficieny, it also can be an issue sometimes. Below we have a section, with font size defined as 5px. Inside we have a stacked ul. The entire ul is meant to be 2em. Let's check how this pans out.

    • Item 1
      • Item 1.1
        • Item 1.1.1
        • Item 1.1.2
        • Item 1.1.3
      • Item 1.2
      • Item 1.3
    • Item 2
    • Item 3
    • Item 4

    We can clearly see that each child is relative to the parent and in this case, while the outer list is 10 px, each inner is doubling on it. This is one disadvatage of em when using with stacked elements.

    This is where rem comes into the picture. rem means root em. So, now instead of inheriting the size from documen root. The document root can be declared by using:

    html{
    font-size: 10px;
    }
    and now by using rem, we can make the dimensions relative to the root. This also helps in terms of the standardization.

    Note: While em and rem are both great, these are mostly used together. To inherit the inner-content size, use rem, so as to keep it proportional and to now declare other elements that are dependent on the inner content use rem.
